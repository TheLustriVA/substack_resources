GitHub Pull Request Cheat Sheet

Step-by-Step Guide

1. Fork and Clone (for external contributions)

# Fork via GitHub web UI, then:
git clone https://github.com/<your-username>/<repo-name>.git
cd <repo-name>

If working in your orgâ€™s repo, just clone as usual.

2. Add Upstream Remote (if contributing externally)

git remote add upstream https://github.com/<original-owner>/<repo-name>.git
# To sync later:
git fetch upstream
git rebase upstream/main   # (or main/master)

3. Create a Feature Branch

git checkout -b my-feature-branch

Example: fix/login-bug or feat/dark-mode-support

4. Make, Add, and Commit Changes

# Make edits, then:
git add .
git commit -m "Clear and concise commit message"

5. Sync Your Branch (before pushing) (recommended)

git fetch upstream
git rebase upstream/main

Resolve any conflicts locally before pushing.

6. Push Your Branch to GitHub

git push origin my-feature-branch

7. Open a Pull Request (PR) on GitHub

  • Go to your repo on GitHub.
  • Click Compare & pull request or use the Pull Requests tab.
  • Set base (upstream main or dev) and compare (your branch).
  • Add a clear Title and Description (link issues: Fixes #123).
  • Assign reviewers/labels if needed, or mark as Draft if not ready.

8. Respond to Reviews

  • Address requested changes; push new commits or rebase as needed.
  • Revise PR description if scope changes.

Pull Request Best Practices

  • One feature or fix per PR (no mixed changes).
  • Clear titles and descriptions; use Fixes #123 to link issues.
  • Proofread all messages; build and test before pushing.
  • Update/add tests as needed.
  • Follow project conventions and naming patterns.
  • Engage on feedback; keep your branch up to date.

Optional: GitHub CLI PR (for terminal junkies)

gh pr create --base main --head my-feature-branch --fill
